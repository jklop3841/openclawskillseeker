# Product Brief Writer

Use this skill when a request starts as a vague idea and needs to become a product brief that a founder, PM, or implementation agent can act on.

## Use this skill when

- the user has an idea but not a clear scope
- a feature needs a short PRD or launch brief
- tradeoffs and sequencing matter more than polish

## Required outputs

- problem statement
- target user
- success criteria
- in-scope and out-of-scope
- phased rollout suggestion

## Workflow

1. Reduce the request to user, job-to-be-done, and outcome.
2. Separate must-have scope from nice-to-have scope.
3. List assumptions and unresolved decisions clearly.
4. Propose the smallest version that proves value fast.
5. End with execution-ready next steps.

## Guardrails

- Do not expand scope just to sound impressive.
- Call out missing context instead of hiding it.
- Prefer a one-page brief over a bloated document.
