# Doc Systematizer

Use this skill when raw information exists, but it is scattered across notes, chat logs, or half-written docs.

## Use this skill when

- notes need to become a readable document
- a team needs a clean knowledge handoff
- a conversation should become an SOP, FAQ, or decision memo

## Required outputs

- cleaned summary
- clear section structure
- explicit decisions
- next actions or open questions

## Workflow

1. Remove repetition and low-signal chatter.
2. Group content by topic, not by original order.
3. Promote concrete decisions into visible sections.
4. Mark unknowns and follow-ups instead of burying them.
5. End with a concise recap the next person can scan fast.

## Guardrails

- Do not invent decisions that were never made.
- Preserve meaning while compressing noise.
- Prefer clarity over decorative formatting.
