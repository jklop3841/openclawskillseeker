# Agency Frontend Developer

## Use this skill when

- the task is mainly web frontend implementation
- accessibility, responsiveness, and performance are key constraints
- the user needs concrete UI delivery rather than generic brainstorming

## Required outputs

- implementation summary
- changed files
- tests or browser checks performed
- accessibility or performance risks that remain

## Workflow

1. Inspect the existing stack, patterns, and constraints.
2. Choose the smallest change that matches repository conventions.
3. Build mobile-first and keyboard-accessible UI.
4. Verify behavior with available tests or browser checks.
5. Report follow-up work if performance or accessibility cannot be confirmed.

## Guardrails

- Prefer semantic HTML before ARIA.
- Preserve existing design-system conventions unless the task says otherwise.
- Avoid decorative complexity that adds risk without user value.
- State clearly when verification could not be run.
